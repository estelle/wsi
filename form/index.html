Accessible Forms

  1. Reading and navigation order must be logical and intuitive.
  2. Must be understood and operated with the keyboard alone (no mouse)
  3. Every form control MUST have a label.
    • Implicit or explicit
    • A button's label is generally it's content
    • Good for audio output, dexterity, and cognition
  4. Put instructions at the beginning of the form, or use aria-describedby on controls.
    • Keyboard users navigate forms with the Tab key. Labels are announced when form inputs receive keyboard focus. Other text is generally skipped.
  5. Associate form controls by using <fieldset> and <legend>.
    • Each checkbox and radio buttons has their own label. If there's more than one, info is needed! if there is only one, do NOT use fielset. Do not nest fieldsets.
  6. optgroup is a good idea, but don't rely on it being read to the user
  7. Don't use multi-select. Use checkboxes if the user can select more than one.

Accessible Forms

  1. The accessible name of a button is it's contents . Don't use a label. Don't leave a button empty!
  2. The accessible name of input of types reset, button, and submit is the value. If no value is present, it's UA text for sumbit, "reset", and inaccessible for button type .
  3. Reset buttons? Just don't.
  4. Image buttons need an alt attribute, rather than a label.
  5. Don't auto-submit forms.
  6. Use autocomplete - helps with congitive issues
  7. Asterisk isn't enough for required fields: put in label or use required attribute (or aria-required)
  8. aria-invalid="true" causes screen readers to announce "invalid" on focus

First rule of ARIA use

ðŸ”—
you can use a native HTML element or attribute with the semantics and behaviour you require already built in, instead of re-purposing an element and adding an ARIA role, state or property to make it accessible, then do so.

HTML is by default accessible. It's our job, as developers, to not fuck that up.

function validate() {
  var input = document.getElementById('b');
  var validityState_object = input.validity;
  if(validityState_object.valueMissing) {
     input.setCustomValidity('Please set an age (required)');
  } else if (input.rangeUnderflow) {
    input.setCustomValidity('Your value is too low');
  } else if (input.rangeOverflow) {
    input.setCustomValidity('Your value is too high');
  } else {
    input.setCustomValidity('');
  }
}
